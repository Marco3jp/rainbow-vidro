# [Phase1][C-13] ボスコアブロック (終盤出現・勝利判定)

## 背景

敵HPが特定閾値以下になると「ボスコアブロック」が出現し、これを破壊することで勝利になる ([`docs/game-design.md`](../docs/game-design.md) §2.3, §2.4)。

## ゴール

- 敵HP閾値で `kind: 'bossCore'` のブロックが1つ生成される。
- ボスコアブロックの破壊で `WorldState.phase = 'won'` に遷移する。

## 作業内容

1. `src/core/systems/bossCore.ts` を作成する。
   - 敵HPが `state.config.bossCoreHpThreshold` 以下になり、まだボスコアブロックが存在しないなら生成
   - ボスコアブロックは特別なHPを持ち、破壊難度を高めに設定 [要調整]
   - 生成位置はフィールド中央上部
2. `winLoseCheck` を更新し、ボスコアブロックの破壊で `phase = 'won'` に遷移するようにする。
3. テスト:
   - 敵HPが閾値以下でボスコアが1つだけ生成される (重複生成しない)
   - ボスコアブロック破壊で `phase === 'won'`
   - 既に `won` のときに更に処理が走らない

## 受け入れ条件

- [ ] テストが緑。
- [ ] 実機で「ボスを削る → ボスコア出現 → 破壊 → 勝利」の流れが動作する。
