# [Phase5 概要] キャラクター・敵の追加

> **NOTE**: 概要のみ。Phase 4 完了後に詳細 Issue へ展開する。

## 背景

[`docs/roadmap.md`](../docs/roadmap.md) Phase 5 のスコープ: キャラクター・敵のバリエーションを追加する。

## 含まれる主なテーマ

- 新キャラクターの追加 (スキル構成・パッシブ・アルティ)
- 新敵の追加 (HP・スキル・出現演出)
- バリエーションを支えるためのデータ駆動設計の整理
  - キャラ・敵を JSON / TS データで定義
  - スキルの効果を組み合わせ可能にするための DSL or テーブル化
- アンロック・選択画面 UI

## 留意点

- データ駆動になることで Phase 1 の system 設計に対する見直しが必要 (パラメータ化が不足している箇所のリファクタ)
- バランス調整のためのチューニングシート (試合ログ集計) の必要性
