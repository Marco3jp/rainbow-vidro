# ゲームデザイン

このドキュメントは「現時点で確定した仕様」と「実装しながら調整する仕様」を区別して記述する。**[要調整]** が付いているパラメータは実装と試遊によって決定される。

## 1. ゲーム全体構造

- 1プレイ ≒ 10分
- フィールド: 16:9 画面のうち、おおむね 4:3 (または 3:4) の領域。残りは情報表示
- 進行: プレイヤーが下側、ブロックが上から徐々に前進
- 終了条件:
  - **勝利**: 「ボスコアブロック」を破壊する (敵HPがゼロになる)
  - **敗北**: プレイヤーHPがゼロになる

## 2. プレイフィールド要素

### 2.1 ボール

- 初期数: 1個
- 速度: 基礎値 **[要調整]**。スキルやギミックで変動
- 消失しない: フィールドから出ない (4辺すべてで反射)
- 衝突: ブロック・バー・壁
- 攻撃力: プレイヤーの攻撃力 × 各種倍率 (反射倍率、チャージ倍率、パッシブ等)

### 2.2 バー / スリング

#### 通常状態

- マウスX (および将来的にY) 追従でフィールド下部 (将来的には他方向も) に位置する
- バーに当たったボールは反射 (反射倍率1倍)

#### 引っ張り (チャージ) 状態

##### 用語定義

| 用語 | 意味 |
| --- | --- |
| **リリース (release)** | チャージしてからマウスを離す操作 |
| **ゼロ位置 (zero position)** | 標準状態の弧位置。チャージ前およびリリース完了後の位置 |
| **チャージ中位置 (charging position)** | チャージ中の弧の現在位置 (ゼロ位置 〜 最大チャージ位置の間) |
| **最大チャージ位置 (max charge position)** | チャージ深さが最大になった時の弧の位置 |
| **リリース位置 (release position)** | リリースした瞬間の弧の位置 (= その時点のチャージ中位置) |
| **ヒット位置 (hit position)** | リリース後の戻り途中で弧がボールに当たった瞬間の弧の位置 |

##### バーの形状変化

- バーは通常時、まっすぐな線分として描かれる (= **ゼロ位置**)
- マウス押下で「チャージモード」に入り、カーソル方向にバーが**弓のように湾曲**する。湾曲の深さは押下時間に比例し、最大で **最大チャージ位置** に到達する
- 押下中はカーソル位置によって湾曲の方向 (= 後の射出方向) が決まる
- マウス押下から最大チャージまでの時間: 基準 200ms **[要調整]**
- リリース後、弧は **リリース位置 → ゼロ位置** に短時間で戻る。戻り時間: 基準 50〜100ms **[要調整]**

##### ボールとバーの相互作用

- ボールは常に通常通り進行する (引っ張り中・戻り中問わず)
- **チャージ中** (`mode === 'charging'`、最大チャージ位置で保持中も含む): ボールが弧 (チャージ中位置 / 最大チャージ位置の弧) に接触すると **その場で停止し、リリース待ちになる**
- **リリース戻り中** (`mode === 'releasing'`): ボールが戻り途中の弧に接触したら **停止せず、その時点でチャージショットとして即射出される** (= 後述の「チャージショット」処理に入る)
- 弧と弧の間 (湾の内側) のボールは通常通り進行する

##### チャージショット (リリーススイープ型メカニクス)

**コアメカニクス**: リリースが起こると、弧は **リリース位置 → ゼロ位置** に時間をかけて戻る (= 弧が空間をスイープする)。**この戻り過程で弧がボールに到達した瞬間、そのボールはチャージショットとして射出される**。

- 戻り中に弧と接触したボール (= ヒット位置にあったボール) は、停止せず、即時に **カーソルが指していた方向 (= チャージ時の引っ張り方向の反対側)** に射出される。射出方向はプレイヤーがチャージ中に行う「引っ張り方向」の操作で決まる
- 射出時のダメージ倍率は次の **2軸の積** で決まる:
  - **チャージ倍率 (`chargeFactor`)**: リリース位置のチャージ深さ。最大チャージ位置でリリース = 最大 (`chargeFactorMax`)、ゼロ位置に近いほど小さい (下限 `chargeFactorMin = 1`)
  - **ヒット倍率 (`hitFactor`)**: ヒット位置 (戻り途中の弧位置) がゼロ位置に近いほど大きく (`hitFactorMax`)、リリース位置に近いほど小さい (下限 `hitFactorMin = 1`)
- チャージショット倍率: `chargeShotMultiplier = chargeFactor × hitFactor` (× キャラの `chargeShotMultiplier` ステータス)
- 暫定値: `chargeFactorMax = 1.2` / `hitFactorMax = 1.2` (両軸ジャストで 1.44 倍) **[要調整]**

##### 保持最大倍率によるクランプ

チャージショットを連続で当てると、ボールの内部倍率に倍率が乗算的に積み重なり青天井になりやすい。これを抑えるため、各キャラに以下のステータスを持たせ、**チャージショット倍率を乗算した直後の値を上限で丸める**。

- `CharacterStats.maxRetainedDamageMultiplier`: 1ボールが保持できるダメージ倍率の上限 (デフォルトキャラの暫定値 `1.5`)
- 適用タイミング: スリングのチャージショット (= `slingControl` / `slingPickup`) の倍率乗算直後
- 例 (`maxRetainedDamageMultiplier = 1.5`、両軸ジャストの倍率 `1.44`):
  - 1度目: 1.0 × 1.44 = 1.44 (上限未満なのでそのまま)
  - 2度目: 1.44 × 1.44 = 2.0736 → 上限 1.5 に丸める
  - 3度目以降: 1.5 × 1.44 = 2.16 ではなく 1.5 に丸める (= 上限で頭打ち)

なお、壁反射時の減衰 (`wallDecayFactor`) や下壁通過時の 1.0 リセットは従来どおり別軸で適用される。

##### リリース時に「弧で停止していた」ボールの扱い

- リリース時点で `attachedBallIds` (= チャージ中に弧で停止していたボール) は、リリース直後に「リリース位置でヒット」した扱いとなり、即時射出される
- ヒット位置 = リリース位置 → `hitFactor` は最低 (`hitFactorMin`)
- ただし `chargeFactor` は `releaseDepth` が高ければ高い (最大チャージ位置で待ってからリリースした場合 `chargeFactor` は最大)
- 射出方向は同じくカーソル方向 (引っ張り方向の反対)
- メリット: ボールを確実に止めて任意のタイミングで射出できる。デメリット: ヒット倍率は最低

##### 戦略性

- 最大威力を出すには、**ボールの速度を予測して、弧がゼロ位置まで戻った瞬間にボールが届くよう** リリースタイミングを調整する必要がある
- 引きが 200ms に対し戻りが 50〜100ms なので、リリース直後に弧がボールへ当たってしまうとヒット倍率は最低
- リリース時点でボールが既に弧に接触して停止している場合、そのボールは即時 (= リリース位置) で射出され、ヒット倍率は最低になる (ただしボールを止めて確実に当てられる戦術メリットあり)

##### 火力ランキング (基礎・通常時)

倍率の概観 (実装値は **[要調整]**):

```
下壁の反射 (1.0 固定)
  < チャージしていないバーの反射 (1.0、ただし別軸の壁減衰の上限)
  < チャージ途中でリリース (chargeFactor × hitFactor、両軸ともグラデーション)
  < 最大チャージ位置でリリース (chargeFactor 最大 × hitFactor グラデーション)
```

最大威力 = 「最大チャージ位置でリリース」かつ「ゼロ位置で当てる」の両方を成立させた時のみ。

#### 反射倍率の概念

| 反射元 | 倍率 |
| --- | --- |
| バー (通常反射) | 1倍 (チャージショットの場合は倍率乗算) |
| 上壁 / 左壁 / 右壁 | 反射ごとに 1倍に向けて段階的に減衰 (例: 1.0 → 0.9 → 0.8 → ... → 1.0 で固定) **[要調整]** |
| 下壁 | 常に1倍 |

これにより「バーで反射する方が有利」というインセンティブを設計する。

#### 下端反射ボールはバーを貫通する

下壁で反射した直後のボールは、次にバー (元の弧 / 引っ張られた弧) に衝突しても通り抜ける。下壁とバーの間でピンポンが発生するのを防ぐため。

- 「下端反射直後フラグ」を `BallState` に持たせ、次の境界 (上/左/右の壁、またはブロック) に衝突するまでバー貫通する
- ブロックや上/左/右の壁に当たる、または十分な時間/距離を移動するとフラグはリセットされる仕様 [要調整]
- 貫通中はチャージショット (吸い込み) の対象にもならない (= 通常進行のみ)

### 2.3 ブロック

すべてのブロックはHPを持つ。HPが0になると破壊され経験値が入る (1stバージョンではゴールドは無し)。

カテゴリ:

| カテゴリ | 内容 | 1stバージョン実装 |
| --- | --- | --- |
| 通常ブロック | 効果なし。経験値のみ | 必要 |
| 特殊ブロック | 周囲ブロック回復、攻撃ショット射出、エリア歪曲などのスキル発動 | 1〜2種類実装 (詳細別Issue) |
| ボスブロック | 通常ブロックよりHPが高く、破壊で「敵」のHPを大きく削る | 必要 |
| ボスコアブロック | 敵HP残量に直結。これを破壊するとゲーム勝利 | 必要 |

#### ブロックの前進

- 連続的かつ固定速度 (基礎値 **[要調整]**)
- 下端到達でプレイヤーへダメージ (固定値 **[要調整]**)
- スキル・ギミックで前進速度を変動可能にする設計

### 2.4 敵 (1stバージョン)

- 1種類のみ
- HPを持つ。ボスブロック・ボスコアブロックの破壊量に応じて減少
- 一定時間ごとに「ボスブロック」を出現させる
- 敵HPが特定閾値以下になると「ボスコアブロック」を出現させる **[要調整]**
- 敵スキルは特殊ブロックの形で発現させる (敵がブロック生成・回復・エリアスキルなどを行う)

## 3. キャラクター (1stバージョン: 1体)

### 3.1 基礎ステータス

| ステータス | 役割 |
| --- | --- |
| HP | プレイヤー体力 |
| マナ | スキル使用コスト |
| 攻撃力 | ボールダメージの基準値 |
| ボール速度 | ボールの基礎速度倍率 |
| スリングパワー (反射倍率, チャージショット倍率) | 反射・チャージ時のダメージ倍率 |
| HP自動回復 | 時間あたりHP回復量 |
| マナ自動回復 | 時間あたりマナ回復量 |
| クールダウン減少 (CDR) | スキルクールダウン低下率 |

### 3.2 スキル構成 (1stバージョン)

「動くゲームができるための最低限のコード網羅」を目的に以下の構成を実装:

1. **パッシブ**: スキルショットが命中するとバフ付与 (ステータスアップ系)
2. **スキル1 (スキルショット)**: 任意タイミングでスリング射出方向に攻撃ショットを撃つ
3. **スキル2 (スリング強化)**: 一時的にバーサイズ拡大 / チャージ倍率上昇 / 貫通付与
4. **アルティメット (ボール複製)**: ボールを一時的に複製

> **NOTE**: 具体的な数値・効果時間・クールダウンは実装中に試遊で調整する。

### 3.3 経験値・レベル

- ブロック破壊で経験値獲得 (種類ごとに値は異なる **[要調整]**)
- 一定値に達するとレベルアップ
- レベルアップ効果:
  - 各ステータスが微増 **[要調整]**
  - スキルポイントを1獲得 → スキル/アルティメット/パッシブのいずれかに振る (LoL方式)
- 1試合内でのレベルアップ。試合終了で初期化

## 4. UI構成 (概要、1stバージョン)

- タイトル画面 (キャラ選択・シード入力・スタート)
- インゲーム画面: フィールド + サイドの情報領域 (HP/MP、スキル一覧、経験値バー、敵HP、経過時間)
- 結果画面 (勝敗、ダメージ、破壊数、リプレイ保存ボタン)
- マッチヒストリー画面 (フェアモード)

## 5. データ・履歴

### マッチヒストリー (1stバージョンで保存する項目)

- 日時
- キャラクター
- レベル / スキルレベルの最終状態
- (将来) 装備アイテム
- ダメージ与えた量
- 破壊したブロック数
- 受けたダメージ
- シード (リプレイ用)
- リプレイログ (オプション、サイズ次第)

保存先: localStorage (1stバージョン)。データ量が増えた場合は IndexedDB への移行を検討。
