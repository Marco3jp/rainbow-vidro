## Phase 1 の作業グループと優先度

「インゲーム体験の楽しさを最優先で検証する」方針に従い、Phase 1 の Issue を作業単位でグループ化し、優先度を 2 段階に区分する。

- **優先度 A**: インゲーム体験を作るために必要。順番に着手していく対象
- **優先度 B**: 後回し (リプレイ・オートプレイ・凝った UI/フォント・E2E テスト等)

GitHub 上では各 Issue は個別に立てるが、作業着手時にはこのグループ単位で1つの作業セッション (= 1 PR) にまとめてもよい。

### 優先度 A (順次着手)

| グループ | 含まれる Issue | 完了時点で得られる状態 |
| --- | --- | --- |
| **G1: プロジェクト基盤** | F-01, F-02, F-04 | Vite/TS/Biome/Vitest/CI が動く |
| **G2: 実行基盤** | F-07, F-08, F-09, F-10, F-11(MVP), F-12 | Pixi 初期化・固定ループ・RNG・Clock・最小Logger・マウス入力が揃う |
| **G3: World とボール・バー・壁反射** | C-01, C-02, C-03, C-04, C-05 | バーをマウスで動かしてボールを反射できる (ブロックなし) |
| **G4: キャラ基礎・ブロック・敗北判定** | H-01, C-06, C-15, C-07, C-08 | ブロックを破壊できる、ブロック下端到達で敗北できる |
| **G5: スリング (湾曲・チャージショット)** | C-09, C-10 | スリング操作のコアメカニクスが遊べる |
| **G6: 試合構造 (ボス・敵スキル・勝利判定)** | C-12, C-11, C-14, C-13 | 1試合を最後まで通せる (時間でボス出現→敵HP削る→ボスコア破壊で勝利) |
| **G7: スキルシステムとキャラスキル一式** | H-02, H-03, H-08, H-04, H-05, H-06, H-07, H-09 | キャラの全スキル4種 + 経験値・レベル・スキルポイント振りが動く |
| **G8: インゲーム最低限UI** | U-01, U-02, U-03 | タイトル → ゲーム → 結果画面 の動線が完成 |
| **G9: 公開・PRプレビュー** | F-05, F-06 | GitHub Pages への本番公開と PR 単位のプレビューが動く |

> G9 は他作業と並行で進められるため、G1〜G3 のいずれかが完了した時点で着手しても良い。

### 優先度 B (後回し、インゲーム検証後に着手)

「インゲーム体験が楽しい」と判断できた後で着手する。

| グループ | 含まれる Issue | 内容 |
| --- | --- | --- |
| **G10: マッチヒストリー** | U-04 | localStorage に試合履歴を保存・閲覧 |
| **G11: フォント・UI整備** | U-06 | Noto Sans JP の本格適用、CSS リセット、レイアウト整備 (※ U-01 で最低限の Noto Sans JP 読み込みは含めるが、整ったタイポグラフィや崩れ対策は U-06 で行う) |
| **G12: リプレイ基盤と再生UI** | F-13, U-05 | 入力ログ記録・保存・再生 |
| **G13: オートプレイ・Fuzz** | Q-01, Q-02 | 簡易戦略によるバグ検出・大量シード Fuzz |
| **G14: ロガー強化** | Q-03 (F-11 の延長) | localStorage 蓄積、ダウンロード UI |
| **G15: E2E テスト** | F-03 | Playwright 導入と最小スモーク |

### グループ間の依存

- G2 完了後に G3 が始められる
- G3 完了後に G4 が始められる (H-01 は C-06 の前提)
- G4 完了後に G5 と G6 のうち G5 から進める方が、スリング体験を早く掴める
- G7 は G3〜G6 と部分的に並行可能。スキル基盤 (H-02, H-03) は G4 完了後でも始められるが、スキルショット (H-05) は C-15 が先にあると安心
- G8 は G3 完了時点から並行で UI を組み始めて構わない (HUD は G7 完了で完成度が上がる)
