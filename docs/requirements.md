# 要件定義

## 1. プロダクト概要

スリング操作を取り入れたブロック崩しベースのWebゲーム。スキルを持つキャラクターを操作し、ギミックを仕掛けてくる敵と戦う。1プレイ約10分のセッションを想定する。

## 2. 想定プレイヤー・体験

- 操作感やテンポの良さ、爽快感
- 敵のギミックへのカウンタープレイの戦略性
- 開始から終了まで画面から目を離せない集中性

## 3. プラットフォーム

| 項目 | 内容 |
| --- | --- |
| 対応プラットフォーム | PCブラウザのみ (Chromium / Firefox / Safari の最新版) |
| スマホ・タブレット | 初期非対応。将来的な対応余地は残すが、初期実装で考慮しない |
| 想定解像度 | 16:9固定。1280×720 / 1440×810 / 1600×900 / FHD のいずれかをCSSスケールで対応 |
| 画面構成 | ブロック崩しフィールド (おおむね 4:3 もしくは 3:4) + 周辺の情報表示領域。比率は実装しながら調整 |
| 言語 | 日本語のみ |

## 4. 機能要件 (1stバージョンスコープ)

詳細は [`game-design.md`](game-design.md) と [`roadmap.md`](roadmap.md) を参照。

- ボール・スリング (バー)・ブロックによるブロック崩しコアループ
- スリングのチャージショット (ジャストタイミング+方向操作)
- 4辺すべてでの反射 (上下左右の壁とバー)、減衰倍率
- ブロックHP、破壊判定、連続前進、下端接触ダメージ
- ボスブロック (時間で出現) → 敵HPに反映 → 終盤に出現する「ボスコアブロック」破壊で勝利
- 敵HP0で勝利、プレイヤーHP0で敗北
- キャラクター1体: 基礎ステータス + スキル2 + アルティメット + パッシブ
- レベル / 経験値: ブロック破壊で経験値、レベルアップでステータス上昇 + スキルポイント
- マナ: 時間経過で回復、スキル使用で消費
- マッチヒストリー保存 (localStorage)
- シード入力によるリプレイ可能性
- オートプレイ (バグ検出用、ヘッドレス実行可能なら望ましい)
- リプレイログ (入力イベント + シード)
- 簡易エラーモニタ (コンソール + localStorage 蓄積 + ダウンロード機能)

## 5. 非機能要件

| 項目 | 要件 |
| --- | --- |
| パフォーマンス | 60Hz以上の安定描画。シミュレーションは固定60Hz、描画は requestAnimationFrame で 120/144Hz もサポート |
| 決定論性 | 同一シード + 同一入力で完全再現可能 |
| アクセシビリティ | 1stバージョンでは特別対応なし (最低限のキーボード操作対応は将来検討) |
| サウンド | 1stバージョン非対応 |
| アセット | 軽量なベクター描画 (PIXI.Graphics または SVG)。パーティクル不使用 |
| フォント | Noto Sans JP (Webフォント) |
| データ永続化 | localStorage。データ量増加時にIndexedDBへの移行を検討 |

## 6. 制約

- ライセンス: MIT
- 課金システム不要 (将来モードでも通貨はゲーム内で獲得)
- デプロイ先: GitHub Pages
- 静的ビルド (サーバサイド処理不要)

## 7. 範囲外 (1stバージョンで作らない)

- アイテムブロック / インゲームアイテム / ゴールド
- ゲーム外強化 (メタ進行)
- 複数キャラクター・複数敵
- 多言語対応
- モバイル対応
- BGM / SE
- 課金・認証・サーバ側機能

これらの拡張順序は [`roadmap.md`](roadmap.md) を参照。
